# Natural Eye and Lens Interaction within a Parametric Customization Framework

This repository accompanies the paper **"Achieving natural eye and lens interaction based eyeglasses optical fit through a parametric customization approach"**. It provides the representative sets of lens utility events used to describe natural eye and lens interaction, together with visualization code for inspecting the spatial and temporal characteristics of the data.

## Overview

Eyeglasses optical performance is commonly evaluated using lens-centered assumptions. In contrast, this dataset represents how the eye actually uses the lens during natural viewing tasks. The deposited data focus on representative lens utility events, which combine:

- **Spatial information**: the location of lens utility points on the lens plane.
- **Temporal information**: fixation-duration-related weighting for each lens utility event.
- **Group information**: representative interaction patterns for the myopia and presbyopia groups.
- **Task context**: natural viewing scenarios such as free walking and digital reading.

The dataset can be used to support optical-fit evaluation, spectacle lens design, eyewear design, and related visual-performance studies where realistic eye–lens interaction is relevant.

## Data Description

The main data correspond to representative sets of lens utility events. In the manuscript, a lens utility event is defined as:

```text
(U, t)
```

where:

- `U` is the spatial lens utility location, representing the region of the lens actually used during a fixation.
- `t` is the corresponding fixation-duration-related temporal value.

Depending on the released data format, each record may include variables such as:

| Field | Description |
| --- | --- |
| `x` | Horizontal coordinate of the lens utility point on the normalized or projected lens plane |
| `y` | Vertical coordinate of the lens utility point on the normalized or projected lens plane |
| `t` | Fixation-duration-related temporal value or representative duration |
| `group` | Participant group, such as myopia or presbyopia |
| `task` | Viewing task or task subset, if provided |
| `density` | Density or visualization weight, if provided |

Please check the released data files for the exact column names and units. The manuscript provides the full methodological details for data acquisition, preprocessing, representative-set construction, and optical-fit evaluation.

## How to Run the Visualization Code

The repository includes Python code for visualizing the representative lens utility event data.

```bash
python code/main.py
```

If the main script has a different filename in the deposited version, run the corresponding visualization script in the `code/` directory.

The script is expected to:

1. Load the representative lens utility event data from the `data/` directory.
2. Plot the spatial distribution of lens utility points.
3. Apply temporal or density-based visual encoding, where available.
4. Save or display the resulting visualization.

## Example Use Cases

This dataset may be useful for researchers and designers working on:

- Spectacle lens design and oblique-aberration evaluation.
- Eyeglasses frame customization and optical-fit assessment.
- Progressive addition lens corridor design.
- Virtual reality, augmented reality, or eyewear display design.
- Human-centered visual-performance evaluation under realistic viewing behavior.

The representative lens utility events provide a practical starting point for evaluating how an eyewear or lens design performs under realistic eye–lens interaction patterns. Researchers may also adapt the pipeline to specific populations, tasks, lens types, or visual scenarios.
